# Domo Magic ETL 解析に基づくフロー機能開発計画（精選版）

| 項目 | 内容 |
| --- | --- |
| 更新日 | 2026-07-13 |
| 方針 | フローチャート編集に本質的なものだけ残す。ETL／共同編集／大規模運用向けの付帯機能は対象外 |

---

## 省いたもの（フローチャート本質外）

| 項目 | 省く理由 |
| --- | --- |
| Sticky notes / 注釈 | 共同レビュー用。図の構造編集に不要 |
| Soft disable（コメントアウト） | ETL デバッグ用途 |
| Select upstream / downstream | 大規模パイプライン向けパワーユーザー操作 |
| キャンバス検索・カタログ検索 | 業務フローは数十ノード規模が主で必須でない |
| Inputs タブ相当の接続ピッカー | 既存のハンドル結線で足りる |
| Copy / Paste | 便利だが必須ではない |
| 複数ページ分割 | 大規模耐性。まずは単一キャンバスで編集可能にする |
| パネル個人設定・Dark Mode | 作業環境の好み。編集機能ではない |
| 真の並列 fork/join | 高度セマンティクス。現状の分岐で足りる段階 |
| バージョン履歴 UI / スナップショット一覧 | 永続化インフラ。キャンバス編集そのものではない |
| AI 生成本番化 | 別トラック（編集 UX とは独立） |

---

## 残す本質機能（実装対象）

業務フロー図を「見て・直して・確定する」ために必要なものだけ。

| ID | 機能 | 理由 |
| --- | --- | --- |
| E1 | ステップ詳細パネル | ラベル以外（担当・システム・項番・種別）を編集できないと図が完成しない |
| E2 | レーン管理 UI | スイムレーン図の前提。追加・改名・削除ができないと構造を変えられない |
| E3 | エッジラベル編集 | 分岐の「はい／いいえ」等を修正できないと条件フローにならない |
| E4 | グラフ検証＋Errors | 開始／終了／孤立／分岐欠落を確定前に検知しないと壊れた図が下流に流れる |
| E5 | 複数選択 | 複数ステップの一括削除・選択操作は基本編集の一部 |

---

## 実装スコープ

### E1. ステップ詳細パネル

- 単一ノード選択時に右サイド（モバイルはボトムシート）で編集
- 項目: ラベル、種別、担当レーン、利用システム、項番
- 変更は `manual: true` 付与、Undo 対象

### E2. レーン管理

- 担当チーム軸で追加／改名／削除
- 削除時: 所属ノードがある場合は移動先レーンを選ばせる
- 改名時: 所属ノードの `data.lane` を同期し `autoLayout` / スナップ座標を更新

### E3. エッジラベル編集

- エッジ選択時にインスペクタでラベル編集
- `enrichEdges` は手動ラベルを尊重（decision 自動補完は空のときのみ）

### E4. グラフ検証

- ルール: start 数、end 到達性、孤立ノード、空ラベル、decision の出力欠落
- Errors パネル（クリックでフォーカス）
- 不正エッジは破線／警告色
- 確定時に error があれば確認ダイアログ

### E5. 複数選択

- Shift＋ドラッグで矩形選択、Shift＋クリックで追加選択
- ツールバーに選択数表示、一括削除は既存 `deleteSelected` を利用

---

## Domo から移植する設計原則（抽象のみ）

1. キャンバス＋設定パネルの二層編集
2. 未完成状態の可視化（破線・エラー一覧）
3. 基本的な複数選択による一括操作

ETL 固有（Run to Here、スキーマ、SQL 変換、タイル無効化等）は移植しない。
