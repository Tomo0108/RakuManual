# ラクマニュアル(プロトタイプ)

業務マニュアル自動作成AIツール「ラクマニュアル」の動作検証用プロトタイプです。
`docs/要件定義書.md` の仕様①〜⑧に対応するUIを、API接続なし(仮データ+モックAI)で実装しています。

## 起動方法

リポジトリルートから `npm run dev`、または:

```bash
cd app
npm install
npm run dev
```

http://localhost:5173/ を開いてください。

## 技術構成

- Vite + React 19 + TypeScript
- Tailwind CSS v4 + shadcn/ui(白基調・アクセントカラー切替可能)
- @xyflow/react(フロー図エディタ)
- フォント: Inter Variable + Noto Sans JP Variable
- ブランド: `Rakumanual.jpg` のロゴを角丸でファビコン・サイドバー・AIアバターに使用。デフォルトのアクセントカラーはロゴに合わせたレッド(設定で5色切替可能)

## 検証用の仮データ(パターン分け)

| プロジェクト | ステータス | 検証できること |
| --- | --- | --- |
| P-001 経費精算業務マニュアル | 公開済み | 完成状態の全画面。承認済みセクション、画像の展開/折りたたみ、手動修正保護フラグ、更新履歴、エクスポート |
| P-002 新入社員PC受入マニュアル | マニュアル作成中 | 「要確認」マークの解消フロー(全解消まで承認不可)、深掘りの状態混在(完了/回答中/未着手/要確認)、プレースホルダセクション |
| P-003 月次請求書発行マニュアル | フロー図編集中 | 生成直後のフロー図編集(差戻しループ含む)、フロー確定→深掘り開始の遷移 |
| P-004 顧客問い合わせ一次対応 | ヒアリング途中 | 中断からの再開、スキップ/わからない/後で答える、回答の修正 |
| 新規作成 | — | 3クリック以内でヒアリング開始 → フローAI生成 → 深掘り → マニュアル生成の一連の流れ |

## 主要機能と対応する要件

- **F-1 骨組みヒアリング**: チャットUI・進捗バー・選択肢/複数選択回答・回答一覧からの修正
- **F-2/F-3 フロー図生成・編集**: ドラッグ&ドロップ移動、ダブルクリックでその場編集、○同士のドラッグで結線、無制限Undo/Redo(⌘Z / ⇧⌘Z)、自動整列、生成根拠のトレース表示、手動修正フラグ
- **F-3 自然言語修正**: 画面下部の入力欄に「AとBの間に◯◯を追加」「◯◯を削除」「◯◯を△△に変更」等と入力 → 差分プレビュー(緑=追加/赤=削除/黄=変更)→ 承認して反映
- **F-4 深掘りヒアリング**: ステップ別の状況一覧、重要度による質問数の変化(高5問/中3問/低1問)、順不同回答
- **F-5 マニュアル生成**: セクション単位生成、「要確認」マーク(全解消まで承認不可)、画像のヘルプボタン方式(本文レイアウトを崩さない)
- **F-6 更新・メンテナンス**: ブロック単位の直接編集、セクション単位のAI再生成、更新履歴、見直し期限
- **F-7 出力・活用**: PDF/PPTX出力設定(テンプレート・範囲・画像の扱い)、業務QAチャット(出典必須・根拠なし回答の拒否・フィードバック収集)
- **F-8 品質測定**: KPIダッシュボード(工数削減率・CSAT・LLMコスト監視など)

## QAチャットの検証パターン

- 「経費精算のやり方を教えて」→ 出典リンク付き回答
- 「請求書の発行手順は?」→ 作成中マニュアルの案内
- 「有給休暇の申請方法は?」→ 該当なし(推測で回答しない)
